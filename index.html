Why use WordPress?

So why would I use WordPress if I like the toolchains of static sites so much? It really came down to images. Basically, the way you manage content on a static site is by committing files directly to a Git repository. Though there are some options available, most of the easy-to-use solutions like Contentful, were fairly expensive. So with all those factors considered, I decided to bite the bullet and make a WordPress site!

Creating an Isolate Environment with Docker

With WordPress selected, I needed to figure out my local development environment. I always found this part intimidating.   • launch ‚Äì Setup the docker container using docker-compose

  • teardown ‚Äì Stop and remove all volumes associated with the site. Useful in case things get too out-of-hand and you want to start from scratch

With these, I was able to create an isolated WordPress environment that I could setup and teardown at will!

Setting Up Live Development

The next thing I needed to figure out was how to make changes to the site. My solution was to create 2 folders at the root of the environment, plugins and themes. The contents of these folders are .gitignored, so I could just git clone existing themes and plugins into them. Then, I created a new npm run script to copy those files into the container:

  • build ‚Äì copies the contents of plugins and themes into the appropriate location in the Docker container

Still, this wasn‚Äôt great for getting immediate feedback. I wanted to be able to copy files into the container as I was editing them. Fortunately, I was able to do that with Gulp.   • dev ‚Äì runs build to copy the initial files over, then runs gulp to watch for changes.

With that, I was able to see changes in real-time as I was making them! This solution might not scale if I need to start modifying other types of WordPress files, but it suits my needs for now. You can see the final environment setup in my comic-site-env repo.
